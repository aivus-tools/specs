# Triage Labels → ClickUp statuses

Лейблов в ClickUp-листах нет, есть статусы. Пять канонических triage-ролей из скиллов Покока маппятся на статусы. Маппинг разный для Dev plan (Conceptualization/Development) и для Bug Tracking.

## Conceptualization / Development

| Роль в mattpocock/skills | Статус                  | Смысл                                              |
| ------------------------ | ----------------------- | -------------------------------------------------- |
| `needs-triage`           | `to do`                 | Заведено, ждёт оценки                              |
| `needs-info`             | `blocked`               | Ждём информацию/решение, двигаться нельзя           |
| `ready-for-agent`        | `to do`                 | Полностью описано, агент берёт в работу (AFK)       |
| `ready-for-human`        | `to do` + комментарий   | Нужна работа человека (пометить в комментарии)      |
| `wontfix`                | `complete` + комментарий | Не делаем (закрыть с пояснением)                    |

Ревью PR и правки по нему — это `in review` → `revisions` → снова `in review` → `complete`.

## Bug Tracking

| Роль / состояние         | Статус             | Смысл                                              |
| ------------------------ | ------------------ | -------------------------------------------------- |
| новый баг                | `Open`             | Только пришёл (через форму или вручную)            |
| `needs-triage`           | `triage`           | Оцениваем Severity, Source, Report Type, Feature   |
| `needs-info`             | `need info`        | Ждём данных от репортера                            |
| в работе                 | `in progress`      | Чиним (bug-fix-protocol: фикс + регресс-тест)       |
| на проверке              | `testing`          | Фикс готов, проверяем                               |
| `wontfix` / не воспроизв.| `not a bug` / `cannot reproduce` | Закрываем без фикса                  |
| исправлено               | `Closed`           | Готово, репортеру можно сообщить                    |

Подтверждённый дефект, требующий заметной работы, заводит задачу в `development` и линкуется через поле Defect Task.
