# 🏗️ ЭТАП 0: АРХИТЕКТУРА И МИГРАЦИЯ BACKEND

**📅 Длительность:** 4 недели (9 октября - 6 ноября)

## Задачи:

### 1. Django Backend
- Создание Django проекта с правильной структурой
- Настройка PostgreSQL и миграции
- Создание всех моделей данных (Users, Vendors, Clients, Briefs, Offers, Categories, Entries, Rates)
- Разработка API endpoints (точная копия текущих):
  - Authentication (register, login, password reset, Google OAuth)
  - Users management
  - Vendors & Clients
  - Categories & Entries
  - Briefs
  - Offers
  - Rates (создание, редактирование, форкинг)
- Реализация HMAC аутентификации и role-based permissions

### 2. Deployment
- Docker контейнеризация
- Helm Charts для Kubernetes
- CI/CD pipeline (GitHub Actions)
- Настройка серверов (staging + production)
- SSL сертификаты
- Nginx конфигурация

### 3. File Storage
- Интеграция S3/MinIO для файлов
- API для upload/download

### 4. Testing
- Pytest setup
- Unit и E2E тесты
- Проверка совместимости с фронтендом

## Результат:
✅ Работающий Django backend на production серверах  
✅ Текущий фронтенд работает без изменений  
✅ Все существующие функции работают  
✅ CI/CD pipeline настроен

# 🛠️ ЭТАП 1: КАБИНЕТ ВЕНДОРА (ДОРАБОТКА)

**📅 Длительность:** 3 недели (7 ноября - 27 ноября)

## Задачи:

### 1. Custom Options в Estimation Table
- Backend: модель EntryOption, API endpoints
- Frontend: UI для создания/выбора опций
- Автоматический пересчет стоимости при выборе опции
- Отображение в сводке

**Результат:** Вендор может добавлять варианты для каждой позиции (например: Sony A7R IV, Canon EOS R5)

### 2. Templates (Шаблоны смет)
- Backend: модель OfferTemplate, API для CRUD
- Frontend: страница Templates
- Сохранение текущей estimation как шаблон
- Применение шаблона к новому offer
- Preview и редактирование шаблонов

**Результат:** Вендор может переиспользовать типовые сметы

### 3. Excel Export
- Backend: генерация Excel файлов (openpyxl/xlsxwriter)
- Frontend: кнопка Export, диалог с опциями
- Форматирование: таблица с категориями, позициями, ценами, сводкой

**Результат:** Скачивание estimation в формате Excel

### 4. Public Links (Публичные ссылки)
- Backend: модель PublicLink, публичные endpoints
- Frontend: генерация ссылок, управление
- Публичная страница просмотра estimation (read-only)

**Результат:** Вендор может делиться estimation через ссылку

# 👔 ЭТАП 2: КАБИНЕТ КЛИЕНТА (СОЗДАНИЕ)

**📅 Длительность:** 4 недели (28 ноября - 25 декабря)

## Задачи:

### 1. Client Dashboard
- Backend: API для статистики дашборда
- Frontend: главная страница клиента
- Карточки статистики (активные проекты, предложения)
- Список проектов с кратким обзором

**Результат:** Клиент видит обзор всех своих проектов

### 2. Brief Creation
- Backend: расширение модели Brief, API для создания
- Frontend: форма создания брифа в 4 шага:
  1. Основная информация (название, описание, deadline, бюджет)
  2. Требования (список с приоритетами)
  3. Deliverables (чек-лист результатов)
  4. Приглашение вендоров

**Результат:** Клиент может создать детальный бриф

### 3. Vendor Invitation Flow
- Backend: модель BriefInvitation, API
- Frontend для клиента: выбор и приглашение вендоров
- Frontend для вендора: получение и обработка приглашений
- Email уведомления

**Результат:** Клиент приглашает вендоров, вендор принимает/отклоняет

### 4. Comparison Table
- Backend: API для сравнения предложений, нормализация данных, расчет статистики
- Frontend: side-by-side таблица сравнения
- Колонки: категории/позиции, цены от каждого вендора
- Цветовая индикация (лучшая цена = зеленый)
- Sticky header, collapse/expand категорий
- Summary карточки по вендорам

**Результат:** Клиент видит все предложения в одной таблице

### 5. Price Analysis
- Backend: анализ цен, outlier detection, best value score
- Frontend: виджеты анализа
- Price insights (средняя цена, отклонения)
- Category analysis (графики по категориям)
- Best value рекомендация

**Результат:** Клиент получает аналитику и рекомендации

# 🔗 ЭТАП 3: ИНТЕГРАЦИИ И ФИНАЛИЗАЦИЯ

**📅 Длительность:** 2 недели (26 декабря - 8 января)

## Задачи:

### 1. Linking Flow
- Backend: модель Project, автоматическое связывание briefs и offers
- Frontend для клиента: страница Projects, управление связями
- Frontend для вендора: страница Projects с статусами
- Ручное добавление offer к brief

**Результат:** Автоматическое и ручное связывание проектов

### 2. AI Integration (Basic)
- OpenAI API интеграция
- Brief Generation: текст → структурированный brief
- Brief Summary: краткое резюме брифа
- Comparison Summary: текстовый анализ сравнения предложений
- Frontend: кнопка "Generate with AI", блок "AI Analysis"

**Результат:** AI помогает создавать брифы и анализировать предложения

### 3. Notifications
- SendGrid интеграция
- Celery + Redis для async emails
- Email templates (HTML)
- Триггеры: новый brief, новый offer, выбор предложения
- In-app notifications (badge, dropdown)

**Результат:** Автоматические email уведомления

### 4. Excel Import
- Backend: парсинг Excel файлов, создание brief
- Frontend: upload и preview

**Результат:** Клиент может импортировать brief из Excel

### 5. Polish & Bug Fixes
- UI/UX улучшения
- Responsive design
- Loading states и error handling
- Form validation
- Performance optimization
- Browser compatibility
- Security audit
- Bug fixes

**Результат:** Готовый к production MVP
